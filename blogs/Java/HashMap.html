# 一文彻底搞懂HashMap

# HashMap的基本概念

HashMap现在可以说是必考的一个集合类。它是用于存储 key-value 键值对的数据结构，一般翻译为“哈希表”。

特点：

  • HashMap是允许null值和null键的。
  • HashMap是非同步的。

# HashMap的静态变量

  1. DEFAULT_INITIAL_CAPACITY
static final int DEFAULT_INITIAL_CAPACITY = 1 << 4; // aka 16
1

HashMap的默认初始容量，规定必须是2的整次幂。这里的默认初始容量是16。

  1. MAXIMUM_CAPACITY
    static final int MAXIMUM_CAPACITY = 1 << 30;
1

HashMap的最大容量，类型为int。规定也必须是2的幂次方，因为int有32位，但最高位是符号位，所以最大容量是2的30次方。

  1. DEFAULT_LOAD_FACTOR
    static final float DEFAULT_LOAD_FACTOR = 0.75f;
1

构造方法中未指定时默认的加载因子，为0.75。

  1. TREEIFY_THRESHOLD
    static final int TREEIFY_THRESHOLD = 8;
1

链表长度大于等于8时，会转换为红黑树。但这只是条件之一

  1. MIN_TREEIFY_CAPACITY
    static final int MIN_TREEIFY_CAPACITY = 64;
1

链表转红黑树的时候，第一步是先检查长度是不是大于等于8，第二步再检查table数组的容量是否小于此数值。如果小于，则取消转红黑树，直接对数组进行扩容。

  1. UNTREEIFY_THRESHOLD
    static final int UNTREEIFY_THRESHOLD = 6;
1

当键值对数量过多时，会对数组进行扩容。并且将每个键值对放到新的桶中。

# 哈希函数

存放数据时，我们都希望元素的位置能尽量分布的均匀一些，尽量使得每个位置上的元素只有一个。

static final int hash(Object key) {
        int h;
        //第一步 取hashCode值
    	//第二步 与自身的高位做异或运算
    	return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    }

//indexfor方法是jdk1.7的源码，jdk1.8没有这个方法，但是实现原理一样的
static int indexFor(int h, int length) {  
    //第三步 取模运算 
    return h & (length-1);
}

这里的hash算法本质上分为3步：

  1. 取h的hashCode值。
  2. 使用扰动函数，对散列值进行扰动，也就是将自己的高16位与低16位做异或。
  3. 做取模运算。

# 为什么做取模运算？

h = key.hashCode()调用的是key键值类型自带的哈希函数，返回的是一个int类型的散列值。

如果直接拿散列值来确定数组索引的话，int的取值范围有40亿那么大，内存是根本放不下的。而且HashMap的初始容量才16，所以这里得对散列值做取模运算。

取模运算是在这个indexFor()函数里完成的。将散列值和数组的长度-1进行与运算。

static int indexFor(int h, int length) {  
    //第三步 取模运算 
    return h & (length-1);
}

为什么这里数组的长度要减1呢？因为HahMap中，数组长度一定要取2的整次幂。因为这样，（数组长度-1）的二进制正好形成了一个低位掩码。比如以初始容量16为例，16-1=15，则15的二进制就为00000000 00000000 00001111。和某散列值进行与操作后，就是截取最低的4位。

	10100101 10101011 00100101
&	00000000 00000000 00001111
---------------------------------
	00000000 00000000 00000101		//高位全部取零，只截取最低4位

# 为什么要加扰动函数？

当散列值只取最后几位的时候，即使散列值分布再松散，哈希碰撞仍然会很严重。

所以扰动函数的作用就体现出来咯，它将散列值自身的高位区和地位区做异或，以此来增加低位的随机性。可以有效地减少碰撞的几率。

# 添加单个元素

//传入键与值，返回值是value。
public V put(K key, V value) {
    // hash(key) 计算哈希值
    return putVal(hash(key), key, value, false, true);
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
               boolean evict) {
    Node<K,V>[] tab; // tables 数组
    Node<K,V> p; // 对应位置的 Node 节点
    int n; // 数组大小
    int i; // 对应的 table 的位置
    
    // <1> 如果 table 未初始化，或者容量为 0 ，则进行扩容
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;
    
    // <2> 通过hash值计算索引位置，将该索引位置的头节点赋值给p。
    //     如果p为空，则直接在该位置新建Node节点即可。
    if ((p = tab[i = (n - 1) & hash]) == null)
        tab[i] = newNode(hash, key, value, null);
    
    // <3> 如果对应位置的Node节点非空，则可能存在哈希冲突，判断各种情况
    else {
        Node<K,V> e; // key 在 HashMap 对应的老节点
        K k;
        
        // <3.1>判断p节点的key和hash值是否跟传入的相等
        //		如果相等，则p节点就为要查找的节点，将p节点赋值给e节点，直接使用
        if (p.hash == hash && //判断hash值相等
            // 判断key真正相等
            ((k = p.key) == key || (key != null && key.equals(k))))
            e = p;
        
        // <3.2> 判断p节点是否为TreeNode节点，是的话直接添加到树中。
        else if (p instanceof TreeNode)
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        
        // <3.3>到这说明p节点是一个普通的链表节点
        else {
            // 顺序遍历链表
            for (int binCount = 0; ; ++binCount) {
                
                //如果遍历完依然没找到key，则直接在尾部新建一个节点
                if ((e = p.next) == null) {
                	p.next = newNode(hash, key, value, null);
                    //检验节点数是否超过8个，如果超过则调用treeifybin()将链表转换为红黑树
                    //-1是因为要算上我们本身插入的元素
                    if (binCount >= TREEIFY_THRESHOLD - 1)
                        treeifyBin(tab, hash);
                    break; // 结束
                }
                // 如果遍历的e节点，就是要找的，则则直接使用即可
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    break; // 结束
                // p 指向下一个节点
                p = e;
            }
        }
        
        // <4.1> 如果找到了对应的节点
        if (e != null) { // existing mapping for key
            V oldValue = e.value;
            // 修改节点的 value ，如果允许修改
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            // 节点被访问的回调
            afterNodeAccess(e);
            // 返回老的值
            return oldValue;
        }
    }
    // <4.2>
    // 增加修改次数
    ++modCount;
    // 如果超过阀值，则进行扩容
    if (++size > threshold)
        resize();
    // 添加节点后的回调
    afterNodeInsertion(evict);
    // 返回 null
    return null;
}

代码虽然有点长，但是逻辑上理一遍就非常简单了。

  1. 如果table未初始化，或者table容量为0，则调用resize()方法，进行扩容。
  2. 再判断传入的key是否为空，如果为空，就存放在table[0]的位置。不为空就对key进行哈希等操作找到桶数组的索引。
  3. 如果对应索引位置的节点为空，则直接新建一个Node节点。这里就相当于一个链表的头节点。
  4. 如果对应位置是非空的话，则可能存在哈希碰撞（也可以叫哈希冲突）。则需要分成两种情况来讨论。

# 扩容机制

// HashMap.java

final Node<K,V>[] resize() {
    //复制一份当前的数组数据，设为oldTab
    Node<K,V>[] oldTab = table;
    //保存旧的数组的阈值和容量
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    //创建两个变量:新容量和新阈值
    int newCap, newThr = 0;
    // <1> 开始：
    // <1.1> oldCap大于0 ，说明table非空
    if (oldCap > 0) {
        // <1.1.1>扩容前数组如果容量已经达到2^30次方，修改阈值为最大值，这样以后就不会再扩容了
        if (oldCap >= MAXIMUM_CAPACITY) {
            threshold = Integer.MAX_VALUE;
            return oldTab; 
        }
        // <1.1.2> newCap = oldCap << 1 ，目的是两倍扩容
        // 如果 oldCap >= DEFAULT_INITIAL_CAPACITY 满足，说明当前容量大于默认值（16），则 2 倍阀值。
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                 oldCap >= DEFAULT_INITIAL_CAPACITY)
            newThr = oldThr << 1; // double threshold
    }
    // <1.2.1>【非默认构造方法】oldThr 大于 0 ，则使用 oldThr 作为新的容量
    else if (oldThr > 0) // initial capacity was placed in threshold
        newCap = oldThr;
    // <1.2.2>【默认构造方法】oldThr 等于 0 ，则使用 DEFAULT_INITIAL_CAPACITY 作为新的容量，使用 DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY 作为新的容量
    else {               // zero initial threshold signifies using defaults
        newCap = DEFAULT_INITIAL_CAPACITY;
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    // 1.3 如果上述的逻辑，未计算新的阀值，则使用 newCap * loadFactor 作为新的阀值
    if (newThr == 0) {
        float ft = (float)newCap * loadFactor;
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                  (int)ft : Integer.MAX_VALUE);
    }
    // <2> 开始扩容：
    // 将newThr赋值给threshold属性
    threshold = newThr;
    // 初始化一个新的Node数组，并赋值给newTab
    @SuppressWarnings({"rawtypes","unchecked"})
    Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
    table = newTab;
    // 如果老的 table 数组非空，则需要进行一波搬运
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            // 取得旧数组中的每个元素
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                // 将旧数组对应位置的元素置空
                oldTab[j] = null;
                // <2.1> 如果e节点只有一个元素，直接将e存到新数组中
                if (e.next == null)
                    newTab[e.hash & (newCap - 1)] = e;
                // <2.2> 如果e节点是红黑树节点，则通过红黑树分裂处理
                else if (e instanceof TreeNode)
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                // <2.3> 如果 e 节点是链表
                else { 
                    Node<K,V> loHead = null, loTail = null;
                    Node<K,V> hiHead = null, hiTail = null;
                    Node<K,V> next;
                    // 这里 do while 的原因是，e 已经非空，所以减少一次判断。细节~
                    do {
                        // next 指向下一个节点
                        next = e.next;
                        // 满足低位
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        }
                        // 满足高位
                        else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    // 设置低位到新的 newTab 的 j 位置上
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    // 设置高位到新的 newTab 的 j + oldCap 位置上
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    return newTab;
}

  1. 计算新的容量和扩容阈值，并创建新的table数组。

# 为什么HashMap是线程不安全的？

  • 多线程同时put或扩容的时候会造成数据丢失。
  • 在1.7版本，扩容方法中会出现死循环造成CPU利用率100%。
  • 1.8版本中，采用尾插法解决死循环问题，但还会造成数据覆盖的问题。
