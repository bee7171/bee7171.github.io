# 2.现实生活的问题场景：租房

  • 房东就是业务类，核心功能就是把房子租出去。额外功能就是打广告，看房。
  • 房客就是调用者。他需要租房。
  • 我们将额外功能外包给一个中介，让他帮忙看房，我们只需要收钱。
  • 如果我们需要新的打广告方式，可以直接找新中介。对比到代码业务中，就大大地降低了耦合。

# 3.代理设计模式

  • 通过代理类，为原始类（目标类）增加额外功能。利于原始类的代码维护。

# 3-1.名词解释

1. 目标类或原始类
		指的是业务类，负责核心功能的类
2. 目标方法或原始方法
		原始类中的方法
3. 额外功能
		事务，日志，性能检测等

# 4.动态代理细节分析

  1. Spring创建的动态代理类在哪里？

    ​ Spring框架在运行时，通过动态字节码技术在JVM创建的，在JVM内部。等程序结束后，会和JVM一起消失。

    **动态字节码：**直接在JVM中生成字节码（动态字节码）

  2. 动态代理编程简化代理的开发

    ​ 在额外功能不变的前提下，创建其他目标类的代理对象时，只需要指定目标对象即可。

  3. 动态代理额外功能的维护性大大增强

# 三、动态代理详解

# 1.MethodBeforeAdvice详解

  • 学习该接口里参数的含义。

# 1.AOP概念

1. AOP(Aspect Oriented Programming)  面向切面过程
		以切面为基本单位的程序开发，通过切面间的彼此协调，相互调用，完成程序的构建
2. OOP 面向对象编程
		以对象为基本单位的程序开发，通过对象间的彼此协调，相互调用，完成程序的构建。
		
		
切面 = 切入点 + 额外功能
