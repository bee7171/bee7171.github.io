# 1.2 实现Runnable接口

实现 java.lang.Runnable 接口也可以创建线程,我们也只需要重写run方法即可。

步骤：

  1. 定义实现Runnable接口的实现类。

  4. 创建Thread类对象，构造方法中传递Runnable接口实现类对象。
  5. 调用Thread类的start方法，开启新的线程执行run方法。

# 1.4 为什么使用Runnable接口更好？

  • 首先从代码架构来看：

实际上，Runnable里只有一个run( )方法，它定义了需要执行的内容，在这种情况下，实现了Runnable与Thread类的解耦，Thread类负责线程启动和属性设置等内容，权责分明。

  • 从Java语言来看：

Java 语言不支持双继承，如果我们的类一旦继承了 Thread 类，那么它后续就没有办法再继承其他的类。这样一来，如果未来这个类需要继承其他类实现一些功能上的拓展，它就没有办法做到了，相当于限制了代码未来的可拓展性。

  • 从性能方面来说：

使用继承Thread类方式，每次执行一次任务，都需要新建一个独立的线程，执行完任务后线程走到生命周期的尽头被销毁，如果还想执行这个任务，就必须再新建一个继承了 Thread 类的类。

如果此时执行的内容比较少，比如只是在run( )方法里简单打印一行文字，那么它所带来的开销并不大，相比于整个线程从开始创建到执行完毕被销毁，这一系列的操作比run()方法打印文字本身带来的开销要大得多，相当于捡了芝麻丢了西瓜，得不偿失。

使用实现 Runnable 接口的方式，就可以把任务直接传入线程池，使用一些固定的线程来完成任务，不需要每次新建销毁线程，大大降低了性能开销。

# 1.6 典型错误观点分析

  • 错误观点一：线程池创建线程也算是一种新建线程的方式。
  • 错误观点二：通过Callable和FutureTask创建线程，也算是新建线程的一种方式。
  • 错误观点三：无返回值是实现Runnable接口，有返回值是实现Callable接口。所以Callable是新的实现线程的方式。
  • 错误观点四：定时器
  • 错误观点五：匿名内部类
  • 错误观点六：Lambda表达式

# 1.7 核心知识点总结

  • 准确地讲，创建线程只有一种方式——构造Thread类。而实现线程的执行单元有两种方式：
    • 方法一：实现Runnable接口的Run方法。并把Runnable实例传给Thread类。
    • 方法二：重写Thread的run方法。
  • 多线程实现的方式在代码中写法千变万化，但其中的本质仍是万变不离其宗。

# 2.2 start()方法原理解读

start()方法的含义：

  • 启动新线程。（本质：请求JVM来运行我们的这个线程）
  • 并不是会立即运行，何时能启动取决于线程调度器的决定。
  • start()方法会让两个线程同时运行。一个是主线程，来执行start()方法。另外一个就是我们创建的线程。

准备工作：

  • 线程执行完该方法会让自己处于就绪状态。已经获取到了除了CPU以外的其他资源。
  • 不能重复执行start方法。

为什么不能重复执行start()方法？

# 3. 停止线程

# 3.1 基本原理

正确停止线程：使用interrupt来通知，而不是强制~！

interrupt是一种合作机制，是让一个线程通知另一个线程停止下来。

被告知该中断的线程具有很高的决定权。它能够决定何时响应中断和停止，还能决定是否停止。

# 3.2 如何正确停止线程？

1. 线程会在什么情况下停止呢？
	run方法所有的代码都运行完毕了。
	有异常出现，且方法中没有捕获。

public class RightWayStopThreadsWithoutSleep implements Runnable{
    @Override
    public void run() {
        int num = 0;
        
        //线程响不响应中断取决于线程本身，所以加上一个响应方法
        //通常情况下要加上isInterrupted方法来告知线程终端才能正常中断。
        while(!Thread.currentThread().isInterrupted() && num <= Integer.MAX_VALUE/2){
            if(num % 10000 == 0){
                System.out.println(num + "是10000的倍数");
            }
            num++;
        }
        System.out.println("任务运行结束了！");
    }

    public static void main(String[] args) throws InterruptedException {
        Thread thread = new Thread(new RightWayStopThreadsWithoutSleep());
        thread.start();
        Thread.sleep(1000);
        thread.interrupt();
    }
}

  • 被弃用的stop，suspend，resume方法。

这种方法已经明确被弃用了。stop方法会导致线程戛然而止，容易造成脏数据。

suspend方法会让线程挂起，但是不会释放锁。所以很容易造成死锁。
