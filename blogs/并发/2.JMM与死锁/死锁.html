# 1.死锁的概念和危害

# 1-1.什么是死锁？

  • 死锁一定是发生在并发中的。
  • 互不相让：当两个（或更多）线程或进程相互持有对方所需要的资源，又不主动释放，导致所有人都无法继续前进。导致程序陷入无尽的阻塞。

# 1-2.多个线程造成死锁的情况

  • 如果多个线程之间的依赖关系是环形，存在环形的锁的依赖关系，就有可能发生死锁。

# 1-3.死锁的影响

死锁的影响在不同的系统中是不一样的。取决于系统对死锁的处理能力。

  • **数据库的措施：**检测并放弃事务。
如果事务1和事务2发生死锁。数据库会检测到，那么他就让事务1主动放弃，这样死锁的问题就解决了。
1
  • **JVM中：**无法处理。

# 1-4.死锁几率不高但危害大

  • 一旦发生，大多都是高并发场景，影响的用户众多。
  • 整个系统崩溃、子系统崩溃、性能降低。
  • 压力测试无法找出所有潜在的死锁。

# 2.发生死锁的例子

# 2-1.最简单的情况

使用一个标记位来实现死锁的情况。

/**
 * 必定发生死锁的情况
 */
public class MustDeadLock implements Runnable{
    //标记位，让两个线程执行不同的方法
    int flag = 1;

    static Object o1 = new Object();
    static Object o2 = new Object();

    public static void main(String[] args) {
        MustDeadLock r1 = new MustDeadLock();
        MustDeadLock r2 = new MustDeadLock();
        r1.flag = 1;
        r2.flag = 0;
        Thread t1 = new Thread(r1);
        Thread t2 = new Thread(r2);
        t1.start();
        t2.start();
    }


    @Override
    public void run() {
        System.out.println("flag = " + flag);
        if(flag == 1){
            //让线程1拿到o1，再去获取o2
            synchronized (o1){
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (o2){
                    System.out.println("线程1成功拿到第二把锁");
                }
            }
        }
        if(flag == 0){
            //让线程2拿到o2，再去获取o1
            synchronized (o2){
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                synchronized (o1){
                    System.out.println("线程2成功拿到第二把锁");
                }
            }
        }
    }
}

# 2-2.实际生产中的例子：转账

  • 需要两把锁，锁住自己的账户和对方的账户，防止同时多个线程对账户操作。
  • 获取两把锁成功，且余额大于0，则扣除自己的钱，增加对方的钱。是一个原子操作。
  • 顺序相反导致死锁。

# 5.如何修复死锁？

  • 线上发生死锁应该怎么办？
1. 线上问题要防患于未然，就像火灾一样，发生了一定会造成损失的。
2. 保存案发线程然后立即重启服务器。
3. 暂时保保障线上服务的安全，然后再利用刚才保存的信息，排查死锁，修改代码。

# 5-1.常见的修复策略

  1. **避免策略:**哲学家就餐的换手方案，转账换序方案。
  2. 检测与恢复策略：一段时间检测是否有死锁，如果有就剥夺某一个资源，来打开死锁。
  3. **鸵鸟策略：**就跟鸵鸟一样，如果发生死锁的概率很低，就忽略它，直到死锁发生了，再人工修复。

通常我们一般考虑避免策略，鸵鸟策略是不推荐的。
