# 深入理解Synchronized关键字

Synchronized关键字的作用就是保证在同一时刻最多有一个线程执行这段代码。以达到并发安全的效果。

# Synchronized的两个用法

# 对象锁

  • 方法锁：默认锁对象为this当前实例对象。
  • 同步代码块锁：自己指定锁对象。

使用代码进行案例演示对象锁的同步代码块形式：

//对象锁的同步代码块形式
public class Demo1 implements Runnable{
    //创建一个实现类对象
    static Demo1 instance = new Demo1();

    @Override
    public void run() {
        //synchronized关键字的代码块形式
        //先用this，即当前对象来作为这个锁
        synchronized (this) {
            System.out.println("我是对象锁的同步代码块形式，我叫： " + Thread.currentThread().getName());
            //休眠3秒钟，方便观察
            try {
                Thread.sleep(3000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(Thread.currentThread().getName() + "运行结束。");
        }
    }

    public static void main(String[] args) {
        //main函数新建两个线程，并启动线程
        //线程中要指定Runnable实现类对象
        Thread t1 = new Thread(instance);
        Thread t2 = new Thread(instance);
        t1.start();
        t2.start();
        
        //两个线程都执行完毕后结束
        while (t1.isAlive() || t2.isAlive()){

        }
        System.out.println("finished");
    }
}

    }

    public static void main(String[] args) {
        Thread t1 = new Thread(instance);
        Thread t2 = new Thread(instance);
        t1.start();
        t2.start();
        while (t1.isAlive() || t2.isAlive()){

        }
        System.out.println("finished");
    }
}

# 类锁

  • Synchronized修饰静态的方法。
  • 指定锁为Class对象。

Java类可能有很多个对象，但只有一个Class对象。

  • 形式1：synchronized加在static方法上。
  • 形式2：synchronized(*.class)代码块。

由于class对象是唯一的，所以所谓的类锁，就是Class对象的锁而已。因为它是唯一的嘛，所以在同一时刻只能被一个对象拥有。

首先来看形式1：加在static方法上的锁。

package concurrency;

public class Demo3 implements Runnable{
    static Demo3 instance1 = new Demo3();
    static Demo3 instance2 = new Demo3();

    @Override
    public void run() {
        method();
    }
    //修饰静态方法
    public static synchronized void method(){
        System.out.println("我是类锁的第一种形式：static形式" + Thread.currentThread().getName());
        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println(Thread.currentThread().getName() + "运行结束！");
    }

    public static void main(String[] args) {
        Thread t1 = new Thread(instance1);
        Thread t2 = new Thread(instance2);
        t1.start();
        t2.start();
        while (t1.isAlive() || t2.isAlive()){

        }
        System.out.println("finished");
    }
}

        t1.start();
        t2.start();
        while (t1.isAlive() || t2.isAlive()){

        }
        System.out.println("finished");
    }
}

# 两个线程同时访问一个对象的同步方法。

# 同时访问同步方法与非同步方法。

synchronized只作用于它修饰的方法，对其他方法没有任何影响。接下来举个例子：

package concurrency;

            method1();

        else{
            method2();
        }
    }

    public synchronized void method1(){
        System.out.println("我是同步方法，我叫" + Thread.currentThread().getName());

        t1.start();
        t2.start();

        }
        System.out.println("finished");
    }
}

# Synchronized的性质

# 1.可重入

**什么是可重入：**指的是同一线程的外层函数获得锁之后，内层函数可以直接再次获取该锁。

**好处：**避免死锁，提高封装性。

# 常见的面试题

# 1.synchronized的使用注意点？

  • **锁对象不能为空。**必须是锁住已经实例好的对象，因为锁是保存再对象头中的，没有对象，何来的锁呢。
  • **作用域不宜过大。**如果把所有代码都放进锁中，无疑会大大降低代码的执行效率。
  • **避免死锁。**死锁会造成程序发生严重错误，所以要尽量避免。
